# Disclaimer Area Resizer — Figma Plugin

Figma-плагин для автоматического расчёта и изменения размера disclaimer-фрейма до нужного процента площади родительского баннера. Если дисклеймера ещё нет, плагин может добавить SVG-дисклеймер из папки `svg/` в текстовую часть баннера или поверх картинки.

## Установка

### 1. Зависимости

```bash
cd disclaimer-area-resizer
npm install
```

### 2. Сборка

```bash
npm run build
```

Команда регенерирует SVG-ассеты, проверит TypeScript и соберёт `dist/main.js` через `esbuild`.

### 3. Подключение в Figma

1. Откройте **Figma Desktop**.
2. Меню **Plugins → Development → Import plugin from manifest...**
3. Выберите файл `disclaimer-area-resizer/manifest.json`.
4. Плагин появится в **Plugins → Development → Disclaimer Area Resizer**.

### 4. Watch-режим (разработка)

```bash
npm run watch
```

Автоматическая пересборка при изменении TypeScript-модулей, импортируемых из `src/plugin.ts`.

## Как пользоваться

1. Откройте баннер в Figma.
2. Выделите **frame с дисклеймером** внутри баннера, если дисклеймер уже есть.
3. Если дисклеймера нет, выделите **сам баннерный фрейм**.
4. Запустите плагин: **Plugins → Development → Disclaimer Area Resizer**.
5. Проверьте размеры баннера и дисклеймера в панели плагина.
6. Выберите **тип дисклеймера** в выпадающем списке.
7. Для добавления нового дисклеймера оставьте чекбокс **Поверх картинки** выключенным, чтобы добавить SVG в текстовую часть. Включите чекбокс, чтобы положить SVG внизу поверх картинки.
8. Чтобы получить набор вариантов, выделите баннер и включите **Создать все типы**. Плагин продублирует баннер для каждого уникального SVG из пресетов и добавит соответствующий дисклеймер.
9. Нажмите **Применить** для существующего слоя, **Добавить** для выбранного баннера без дисклеймера или **Создать** для набора вариантов.

Как плагин находит баннер: он поднимается по цепочке родителей от выделенного слоя и берёт самый верхний фрейм перед страницей.

```
Page
  Banner 1080×1920   ← bannerFrame
    Content
      Disclaimer     ← выделите этот слой для ресайза
```

Для добавления с нуля выделяется сам баннер:

```
Page
  Banner 1080×1920   ← выделите этот фрейм для добавления SVG
    Text content
    Image
```

## Формулы расчёта

```
bannerArea    = bannerWidth × bannerHeight
disclaimerArea = disclaimerWidth × disclaimerHeight
currentPercent = disclaimerArea / bannerArea × 100
targetArea    = bannerArea × targetPercent / 100
```

**Direction = height** (меняем только высоту):
```
newWidth  = disclaimerWidth  (без изменений)
newHeight = targetArea / disclaimerWidth
```

**Direction = width** (меняем только ширину):
```
newHeight = disclaimerHeight  (без изменений)
newWidth  = targetArea / disclaimerHeight
```

**Direction = proportional** (масштабируем равномерно):
```
scale     = √(targetArea / disclaimerArea)
newWidth  = disclaimerWidth × scale
newHeight = disclaimerHeight × scale
```

**Добавление нового SVG**:
```
targetArea = bannerArea × targetPercent / 100
newWidth   = ширина текстового контейнера или медиа-области с учётом padding
newHeight  = targetArea / newWidth
```

SVG можно деформировать по высоте: плагин сохраняет нужную площадь дисклеймера относительно всего баннера.

**Создать все типы**:
- работает только при выборе баннерного фрейма;
- создаёт дубликаты исходного баннера рядом справа;
- берёт по одному основному пресету на каждый уникальный SVG из `DISCLAIMER_PRESETS`;
- добавляет дисклеймер в body или поверх картинки по состоянию чекбокса **Поверх картинки**.

**Пример:**
- Баннер: 1000×1000 = 1 000 000 px²
- Disclaimer: 1000×40 = 40 000 px² = 4%
- Цель: 7% → targetArea = 70 000 px²
- Height:       1000 × 70
- Width:        1750 × 40
- Proportional: ≈ 1322.9 × 52.9

## Пресеты и их настройка

Пресеты находятся в `src/core/presets.ts`, объект `DISCLAIMER_PRESETS`. Каждый пресет содержит процент и `assetKey` — ключ SVG-дисклеймера из `src/generatedDisclaimerAssets.ts`.

Чтобы изменить процент — отредактируйте поле `percent` нужного пресета и пересоберите плагин:

```typescript
const DISCLAIMER_PRESETS: Record<string, DisclaimerPreset> = {
  medicine_video_7: {
    label: "Медицина — 7% / ТВ, видео или по ТЗ",
    percent: 7, // ← изменить здесь
    assetKey: ASSET_MEDICINE,
  },
  // ...
};
```

После изменения:
```bash
npm run build
```

Перезагрузка плагина в Figma происходит автоматически при следующем запуске.

## SVG-дисклеймеры

Исходные SVG лежат в папке `svg/`. Figma-плагин не читает локальные файлы во время работы, поэтому при сборке они встраиваются в TypeScript-файл:

```bash
npm run generate:assets
```

Команда создаёт `src/generatedDisclaimerAssets.ts`. Обычный `npm run build` запускает генерацию автоматически.

Текущий маппинг:

| Пресеты | SVG |
|---------|-----|
| `medicine_video_7`, `medicine_static_5` | `Есть противопоказания...svg` |
| `bad_static_10`, `bad_video_7`, `energy_7` | `Не является лекарством.svg` |
| `finance_credit_5` | `Изучите все условия кредита...svg` |

## Разработка

Runtime плагина собирается из `src/plugin.ts`. Код разложен по модулям:

- `src/core/` — чистая математика, типы и пресеты;
- `src/figma/` — безопасные обёртки над Figma Plugin API;
- `src/features/` — сценарии resize/add/create-all;
- `src/state/` — состояние текущего выделения для UI;
- `src/ui/` — типы сообщений между UI и runtime.

После изменения TypeScript-кода всегда коммитить обновлённый `dist/main.js`. После изменения SVG также коммитить обновлённый `src/generatedDisclaimerAssets.ts`.

## Ограничения

- Плагин считает **геометрическую площадь** bounding box, а не юридическую корректность текста.
- Для статичных и видеоформатов могут быть **разные проценты** — это отражено в пресетах.
- При **auto layout** Figma может скорректировать размер после применения из-за layout constraints родителя.
- Если в макете есть **rotation или effects**, визуальная площадь для зрителя может отличаться от Figma geometry.
- При ресайзе существующего дисклеймера плагин не меняет позицию слоя, размер шрифта, текст, цвета или другие слои.
- При добавлении нового дисклеймера плагин создаёт SVG-слой, помечает его plugin data и выбирает его после вставки.
- В режиме **Создать все типы** исходный баннер не меняется; плагин создаёт дубликаты с разными SVG-дисклеймерами и оставляет выбранным исходный баннер.
- Плагин **не работает** с заблокированными (locked) слоями.
- Плагин не использует сеть и не обращается к REST API Figma.
